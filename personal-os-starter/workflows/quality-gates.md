# Quality Gates for Summary Skills

Apply these checks before finalizing any summary output (meeting recaps, weekly updates, channel posts).

## Gates

1. **Completeness** — Cross-reference output against source material. Every topic/project in the sources must appear in the summary. If sources covered 8 topics, the summary reflects all 8.
2. **Length** — Channel posts: 5-8 bullets per section. DMs: under 300 words. Weekly updates: one line per project.
3. **Separation** — Distinct workstreams are never merged into one bullet. Each project = its own line.
4. **Tone** — Read the appropriate voice guide (`voice/private.md` for DMs/Slack, `voice/public.md` for external). No exclamation points, no corporate jargon.
5. **Formatting** — Follow the Slack Message Style Guide from CLAUDE.md. Use `content_type="text/plain"`, `*` for bullets.
6. **Sensitivity** — Flag anything that could be sensitive (personnel, financials, unreleased strategy) for explicit approval before including.
7. **Self-check** — Before presenting output, re-read the draft against gates 1-6. If any gate fails, revise silently (up to 2 iterations), then present the final version.
